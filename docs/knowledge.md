# Card Draft 개발 메모

## 프로젝트 목표

`Card Draft`는 판타지마스터즈의 종족전 감성과 하스스톤식 접근성을 섞은 온라인 TCG MVP다.

현재 목표는 완성형 온라인 게임이 아니라, Godot 안에서 아래 핵심 재미가 실제로 성립하는지 빠르게 검증하는 것이다.

- 필드 전투
- 종족 시너지
- 속성 정체성
- 덱 빌딩

MVP 철학은 `필드 전투 80%, 카드 효과 20%`다. 복잡한 효과보다 유닛을 내고, 교환하고, 필드를 잡는 판단이 더 중요해야 한다.

## 핵심 룰

- 승리 조건: 상대 영웅 체력 20을 0 이하로 만든다.
- 덱 크기: 최종 룰 기준 30장.
- 동일 카드 제한: 최종 룰 기준 최대 3장.
- 필드 제한: 플레이어당 유닛 최대 5장.
- 전투: 공격 유닛과 방어 유닛이 동시에 피해를 준다.
- 덱 고갈: 현재 구현에서는 드로우 실패 시 영웅이 피해 1을 받는다.
- 일반전은 랭크 점수를 바꾸지 않는다.
- 랭크전은 승리 시 점수 +25, 패배 시 점수 -10을 적용한다.
- 랭크 점수는 0 아래로 내려가지 않는다.

## 로컬 진행 데이터

서버가 꺼져 있으면 `user://profile.json`에 아래 값을 저장한다.

- `gold`: 골드.
- `rank_points`: 랭크 점수.
- `owned_cards`: 카드 ID별 보유 수량.
- `selected_deck`: 카드 ID 30개 배열.
- `settings`: 전투 연출, AI 턴 속도 설정.

첫 실행 또는 초기화 시 모든 샘플 카드를 2장씩 지급하고, 앞쪽 기본 카드 일부는 3장까지 지급해 30장 덱을 바로 구성할 수 있게 한다.

서버가 켜져 있으면 Godot는 `POST /api/guest-login`으로 게스트 유저를 만들고, 이후 `X-User-Id` 헤더로 프로필, 보유 카드, 덱, 매치 결과를 동기화한다.

## 랭크 기준

- 브론즈: 0-499점.
- 실버: 500-999점.
- 골드: 1000-1499점.
- 플래티넘: 1500-1999점.
- 다이아: 2000점 이상.

현재 랭크는 로컬 점수 표시용이며 시즌 초기화, MMR, 승급전은 없다.

## 상점 기준

현재 상점은 게임 내 골드만 사용한다. 실제 과금은 없다.

상품:

- 견습 카드 팩: 골드 50, 전체 카드 중 랜덤 1장.
- 종족 지원 팩: 골드 80, 선택 종족 카드 중 랜덤 1장.
- 전장 보급 팩: 골드 120, 전체 카드 중 랜덤 3장.

중복 카드는 보유 수량으로 3장을 초과할 수 있다. 다만 덱에는 동일 카드 최대 3장만 넣을 수 있다.

## 종족 방향

- 인간: 안정적인 능력치, 아군 버프.
- 엘프: 저코스트 전개, 카드 드로우.
- 언데드: 사망 이득, 희생, 부활 계열.
- 중립: 모든 덱에서 쓸 수 있는 단순 유틸리티.

주의할 점:

- 인간은 너무 싼 광역 버프를 받으면 필드를 한 번에 터뜨린다.
- 엘프는 드로우와 좋은 스탯을 동시에 받으면 템포 손실이 없다.
- 언데드는 자해 비용이 낮으면 리스크 없는 드로우가 된다.
- 중립 카드는 종족 카드보다 강한 핵심 카드가 되면 안 된다.

## 속성 방향

- 화염: 직접 피해.
- 물: 회복.
- 바람: 기동성, 추가 공격, 빠른 전개.
- 대지: 체력 증가, 방어.
- 암흑: 희생, 자해, 사망 가치.
- 빛: 버프, 지원.

## 밸런스 기준

MVP 밸런스 목표는 필드전 중심이다. 주문, 드로우, 장착은 필드 싸움을 보조해야지 게임을 단독으로 끝내면 안 된다.

유닛 스탯 기준:

- 1코스트: 총합 2스탯 기준.
- 2코스트: 총합 4스탯 기준.
- 3코스트: 총합 6스탯 기준.
- 4코스트: 총합 8스탯 기준.
- 효과가 좋은 유닛은 기준보다 총합 스탯을 낮춘다.

효과 기준:

- 2장 드로우는 기본 3코스트.
- 3코스트 피해 4는 가능하지만 영웅 직접 피해는 테스트 우선 대상.
- 2코스트 회복 3은 필드에 영향을 주지 않으므로 허용.
- 전체 공격력 버프는 최소 3코스트.
- 영구 장착 효과는 반복 누적되면 위험하므로 카드 수를 늘릴 때 주의한다.

상세 기준은 `res://docs/balance.md`에 정리되어 있다.

## 카드 데이터 작성 규칙

카드는 `data/cards.json`에 JSON 배열로 작성한다.

필수 필드:

- `id`: 내부 식별자. 영어 snake_case 권장.
- `name`: 표시 이름.
- `type`: `unit`, `spell`, `equipment` 중 하나.
- `race`: `인간`, `엘프`, `언데드`, `중립` 중 하나.
- `attr`: `화염`, `물`, `바람`, `대지`, `암흑`, `빛` 중 하나.
- `cost`: 마나 비용.
- `art`: 카드 아트 시트의 인덱스.
- `text`: 카드 설명.

유닛 추가 필드:

- `attack`: 공격력.
- `health`: 체력.

현재 `main.gd`의 효과 처리는 카드 `id` 기준으로 직접 분기한다. 새 효과 카드를 추가하면 카드 데이터만으로는 동작하지 않고, `main.gd`의 `_resolve_spell`, `_play_card`, 사망 처리 등에 로직을 추가해야 한다.

## 아트 시트 규칙

현재 카드 이미지는 한 장짜리 아트 시트를 잘라 쓴다.

- 파일: `assets/card_art/season1_sample_sheet.png`
- 배열: 4열 x 3행
- 인덱스: 왼쪽 위부터 오른쪽으로 0, 1, 2 순서

예:

- `art: 0`은 첫 번째 칸.
- `art: 4`는 두 번째 줄 첫 번째 칸.
- `art: 11`은 마지막 칸.

현재 샘플 카드셋은 12장을 넘기 때문에 기존 아트 12칸을 여러 카드가 재사용한다.

장기적으로는 다음 중 하나가 필요하다.

- 더 큰 아트 시트로 교체하고 `CARD_ART_COLS`, `CARD_ART_ROWS` 수정.
- 카드별 개별 이미지 파일 로딩 구조로 변경.
