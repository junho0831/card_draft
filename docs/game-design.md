# Card Draft Game Design

이 문서는 현재 코드 기준의 `Card Draft` 게임 기획서다. 구현 기준은 Godot 프로젝트의 `data/*.json`과 `src/**/*.gd`이며, 목표는 온라인 TCG가 아니라 `빌드 완성형 싱글플레이 로그라이크 덱빌딩 RPG`의 한 런을 완주 가능한 형태로 제공하는 것이다.

## 핵심 컨셉

플레이어는 약한 시작 덱으로 런을 시작해 전투, 보상, 이벤트, 상점, 휴식을 거치며 덱과 유물을 강화한다. 게임의 재미는 카드 수집 자체보다 `이번 런이 어떤 빌드로 완성되는가`에 있다.

플레이어가 매 화면에서 알아야 하는 것은 세 가지다.

- 지금 무엇을 해야 하는가.
- 이번 런은 어떤 빌드로 성장하고 있는가.
- 지금 선택하면 덱이 어떻게 강해지는가.

## 현재 콘텐츠 범위

- Act 1개: `국경지대`
- 카드 38종
  - 스타터 카드 4종
  - 런 카드 풀 34종
- 유물 15종
- 이벤트 5종
- 적 13종
  - 일반 적 8종
  - 엘리트 3종
  - 보스 2종

## 런 흐름

기본 흐름은 다음과 같다.

```text
메인 메뉴
-> 새 런 시작 또는 이어하기
-> Act 맵
-> 노드 진입
-> 전투 / 이벤트 / 상점 / 휴식
-> 카드 보상 또는 노드 완료
-> 보스
-> 다음 Act 또는 런 결과
```

Act 데이터는 `data/acts.json`에 있으며, 현재 기본 Act는 5개 레이어로 구성된다. 기본 목표는 한 판을 5-10분 안에 끝내는 짧은 런이다.

```text
1. battle
2. event 또는 shop
3. battle
4. rest 또는 shop
5. boss
```

실제 현재 기본 Act 데이터는 다음 흐름이다.

```text
1. battle
2. event 또는 shop
3. battle
4. rest 또는 shop
5. boss
```

현재 맵은 완전한 트리형 분기라기보다, 일부 레이어에서 플레이어가 현재 선택지를 고르는 구조다. 노드 완료 후에는 다음 레이어로 진행한다.
엘리트는 기본 흐름에서 제외되어 있으며, 후속 도전 모드 후보로 남긴다.

## 시작 상태

기본 시작 덱은 인간 덱이다.

```text
민병대 x3
초보 검병 x3
작은 불꽃 x2
응급 치료 x2
```

기본 시작 체력은 50, 기본 시작 골드는 100이다. 메타 업그레이드가 있으면 시작 체력과 시작 골드가 증가한다.

코드에는 `elf`, `undead` 시작 덱과 시작 유물 함수가 남아 있지만, 현재 UI 흐름에서는 시작 종족 선택을 노출하지 않는다.

## 전투 규칙

전투의 목표는 적 영웅 체력을 0으로 만드는 것이다. 플레이어 영웅 체력이 0이 되면 패배한다.

- 플레이어와 적은 각각 영웅 체력을 가진다.
- 필드는 플레이어 5칸, 적 5칸이다.
- 유닛 전투는 동시 피해다.
- 마나는 1부터 시작하고 턴마다 최대 마나가 1 증가한다.
- 최대 마나는 10이다.
- 시작 손패는 4장이다.
- 턴 시작 시 카드 5장을 드로우한다.
- 손패 제한은 10장이다.
- 턴 종료 시 남은 손패는 버림 더미로 이동한다.
- 덱이 비면 버림 더미를 섞어 새 덱으로 만든다.
- 덱과 버림 더미가 모두 비어 있으면 드로우 대신 영웅이 피해 1을 받는다.

전투 UI는 규칙 설명보다 현재 선택을 돕는 짧은 안내를 우선한다. 플레이어 턴에는 추천 카드나 추천 공격을 한 줄로 보여주고, 적 턴에는 다음 적 행동을 `유닛 공격`, `영웅 공격 위험`, `소환 준비`처럼 표시한다.

저주와 의식은 턴 시작 시 1회성으로 발동하는 상태 스택이다. 저주는 보유한 스택만큼 영웅 피해를 주고 사라진다. 의식은 보유한 스택만큼 영웅 체력을 회복하고 사라진다. 별도 승리 조건은 아니며, 승리 조건은 `적 영웅 체력 0` 하나로 고정한다.

## 전투 리듬과 초보자 흐름

현재 전투 화면은 `규칙 설명`보다 `다음 한 수`를 먼저 보여주는 구조를 따른다. 기본 플레이 흐름은 다음 3단계다.

1. `추천 진행` 버튼이나 손패의 추천 카드부터 누른다.
2. 공격 가능한 유닛이 생기면 카드보다 공격을 먼저 처리한다.
3. 더 할 행동이 없으면 `턴 종료`를 누른다.

전투 상단에는 항상 다음 두 줄이 남는다.

- `지금 할 일`: 지금 가장 먼저 해야 하는 행동
- `이번 턴 플랜`: 카드 사용 -> 공격 확인 -> 턴 종료 같은 짧은 행동 순서

첫 전투에는 짧은 튜토리얼 패널이 뜬다. 여기서는 아래 3가지만 알려준다.

- 추천 진행부터 눌러도 된다.
- 밝은 카드와 공격 가능한 유닛만 먼저 보면 된다.
- 같은 빌드 태그 카드를 연속으로 내면 연계가 발동한다.

플레이어가 한 번 닫으면 이후에는 다시 뜨지 않는다.

## 전투 UI 구현 기준

현재 Godot 전투 UI는 `컨트롤 노드 + 컨테이너 + Tween` 중심으로 구성한다. 목표는 카드 규칙을 길게 설명하는 것이 아니라, `지금 눌러야 할 것`과 `눌렀을 때 터지는 보상감`을 우선 보여주는 것이다.

현재 코드 기준 핵심 구조:

- 전투 루트는 `src/ui/screens/battle_screen.gd`
- 공통 패널/칩/배지/카드 텍스트 스타일은 `src/ui/ui_factory.gd`
- 손패 영역은 `HFlowContainer` 기반
- 전장 유닛은 `Button` 형태의 카드 슬롯으로 구성
- 마우스 반응은 `mouse_entered`, `mouse_exited`와 `create_tween()`으로 처리

현재 이미 반영된 UI 원칙:

- 손패 카드는 사용 가능 여부, 추천 여부에 따라 테두리와 밝기가 달라진다.
- 카드에 마우스를 올리면 확대되고 `z_index`가 올라가 앞쪽으로 튀어나온다.
- 카드 표면은 `fantasy_card_frame.png`, `fantasy_ui_panel.png` 자산을 이용해 이름 띠, 효과 본문, 상태 바를 분리한다.
- 카드 핵심 효과는 공용 요약 문장으로 먼저 보여주고, 원문 설명은 보조 정보로 낮춘다.
- 숫자와 상태 문구는 outline을 주어 배경에 묻히지 않게 한다.
- 연계, 처치, 승리 순간에는 큰 플로팅 텍스트, 화면 흔들림, 사운드가 함께 나온다.

현재 아직 남아 있는 다음 단계:

- 손패를 `HBox/HFlow` 나열형에서 `일반 Control 부모 + 수동 위치/회전 계산` 방식의 부채꼴 손패로 바꾸기
- 카드 드래그 시 손패 재정렬과 목표 슬롯 프리뷰 추가
- 모바일에서는 부채꼴보다 `가로 스와이프 + 중앙 확대 카드`가 더 나은지 별도 검증

즉, 현재 구현은 `카드 가독성 + 호버 반응 + 도파민 피드백`까지는 들어와 있고, 다음 폴리싱 단계는 `손패를 실제로 손에 쥔 듯한 배치`다.

## 보상 규칙

일반 전투 승리 시 카드 보상과 골드를 받는다.

- 일반 전투: 골드 15-25, 카드 3장 중 1장 선택
- 엘리트: 골드 30-50, 카드 보상, 유물 보상
- 보스: 골드 50, 카드 보상, 유물 보상, 최대 체력 +5

카드 보상 3장 중 1장은 현재 최고 빌드 태그와 맞는 카드에서 우선 제시된다. 이를 통해 플레이어가 현재 런의 방향을 유지하거나 강화할 수 있다.
보상 화면은 `현재 빌드에 맞음`, `즉시 전투 도움`, `덱 압축 후보` 같은 짧은 이유를 함께 보여줘 빠르게 선택하게 한다. 추가로 각 카드에는 `빌드 상승` 줄을 붙여, 어떤 빌드 점수가 얼마나 오르는지와 이번 선택으로 활성화가 되는지를 즉시 보여준다.

## 빌드 태그

카드와 유물은 `build_tags`를 통해 빌드 방향을 가진다. 카드 한 장은 태그 점수 1점, 유물 하나는 태그 점수 2점으로 계산한다.

유효 태그는 다음 6개다.

- `fire`: 화염
- `draw`: 드로우
- `death`: 사망
- `buff`: 버프
- `low_hp`: 저체력
- `summon`: 소환

빌드 점수 5 이상이면 해당 빌드가 활성화된다.

활성 효과는 다음과 같다.

- 화염: 화염 피해 +2
- 드로우: 플레이어 턴 시작 시 카드 1장 추가 드로우
- 사망: 아군 유닛 사망 시 적 영웅 피해 1
- 버프: 아군 유닛 소환 시 체력 +1
- 저체력: 내 영웅 체력 50% 이하일 때 내 유닛 공격 피해 +1
- 소환: 전투 시작 시 1/1 중립 토큰 1장 소환

빌드 점수는 맵, 전투, 보상, 상점, 이벤트, 결과 화면에서 반복적으로 보여준다.

## 연계 시너지

현재 구현에는 `빌드 활성`과 별개로, 같은 활성 빌드 태그 카드를 한 턴에 연속으로 사용할 때 발동하는 `연계 시너지`가 들어간다. 목적은 “내가 빌드를 맞춰서 강한 턴을 만들었다”는 체감을 전투 중 바로 주는 것이다.

기본 규칙은 다음과 같다.

- 한 턴 안에서 같은 활성 빌드 태그 카드가 2장 이상 연속으로 이어지면 연계 발동
- 다른 태그 카드가 끼면 연계 흐름이 바뀜
- 턴이 끝나면 연계 카운트는 초기화

현재 태그별 연계 효과는 다음과 같다.

- `fire`: 적 전열 또는 적 영웅에게 추가 피해 2
- `draw`: 카드 1장 추가 드로우
- `death`: 적 영웅에게 추가 피해 1
- `buff`: 가장 앞의 아군 유닛 공격력 +1
- `low_hp`: 내 영웅 체력 1 회복
- `summon`: 마지막으로 소환한 유닛이 즉시 공격 가능

연계가 발동하면 전투 로그와 플로팅 텍스트로 즉시 알린다. 예시는 `화염 연계 2`, `유리한 교환`, `결정타`, `핵심 시너지 점화` 같은 피드백이다.

추가로 현재는 아래 도파민 포인트를 별도 연출로 강화했다.

- 연계 발동: 전용 사운드 + 큰 텍스트 + 글로우
- 처치: `처치!` 피드백 + 강한 화면 흔들림
- 강타: 큰 피해가 들어가면 별도 `강타` 피드백
- 승리 직후: `승리!`, `승리 +골드` 배너와 보상 사운드

## 카드와 효과

카드는 `unit`, `spell`, `equipment` 타입을 가진다.

- 유닛은 필드에 소환되고 공격력/체력을 가진다.
- 주문은 즉시 효과를 처리한 뒤 버림 더미로 간다.
- 장비는 현재 아군 유닛에게 적용된다.
- 카드 이미지는 `art_id`로 `assets/card_art/cards/{art_id}.png`를 우선 로드한다.
- 개별 이미지가 없으면 기존 `art` 시트 인덱스를 fallback으로 사용한다.

대표 카드 효과:

- `small_flame`, `fireball`: 앞의 적 또는 적 영웅에게 피해
- `first_aid`, `healing_potion`: 영웅 체력 회복
- `captain_order`: 아군 전체 공격력 증가
- `dark_bargain`: 체력을 지불하고 카드 드로우
- `call_of_dead`: 해골 병사 소환
- `corpse_explosion`: 아군을 희생하고 적에게 피해
- `bone_soldier`: 사망 시 적 영웅에게 피해 1
- `grave_knight`: 사망 시 내 영웅 체력 2 회복

## 유물

유물은 전투 훅 기반으로 작동한다. 획득 시 즉시 능력치를 바꾸는 유물, 턴 시작/카드 사용/유닛 소환/영웅 피해/승리 보상에 반응하는 유물이 있다.

유물은 빌드 태그를 가질 수 있으며, 빌드 점수 계산에서 카드보다 큰 비중인 2점을 제공한다.

## 이벤트

이벤트는 런 중 선택 변수를 제공한다. 각 이벤트는 설명, 선택지, 비용, 보상을 가진다.

현재 이벤트 결과는 다음 범주의 효과를 포함한다.

- 체력 손실
- 골드 지불
- 카드 보상 생성
- 유물 획득
- 카드 강화
- 카드 제거
- 회복
- 최대 체력 변화

선택 후에는 결과 메시지를 보여주고 현재 노드를 완료한다.

## 상점

상점은 런 중 덱을 조정하는 공간이다.

제공 기능:

- 카드 구매
- 유물 구매
- 회복
- 카드 제거
- 덱 확인

상점 기본 가격은 카드 35골드, 유물 110골드, 회복 45골드다. 카드 제거 비용은 기본 45골드이며, 제거할수록 20골드씩 증가한다. 구매할 수 없는 항목은 비활성 상태로 보여준다.

## 휴식

휴식 노드는 전투 사이의 정비 구간이다.

현재 기능:

- 휴식: 최대 체력의 30% 회복
- 카드 강화: 덱 카드 1장 강화

휴식 또는 강화가 끝나면 노드를 완료하고 다음 레이어로 진행한다.

## 메타 진행

런 종료 후 영혼석을 얻고, 로컬 프로필에 저장한다. 현재 메타 업그레이드는 다음 항목을 다룬다.

- 시작 최대 체력 증가
- 시작 골드 증가
- 두 번째 기회

두 번째 기회는 전투 중 플레이어 체력이 0이 될 때 한 번 체력 1로 버티는 효과다.

## 저장과 이어하기

런 상태는 `user://run_state.json`에 저장된다. 메타/설정/컬렉션 정보는 `user://meta_profile.json`에 저장된다.

이어하기는 다음 상태를 복원한다.

- 현재 Act와 노드
- 덱, 유물, 체력, 골드
- 진행 중인 전투
- 전투 손패, 필드, 덱, 버림 더미, 턴, 로그
- 카드 보상
- 이벤트
- 상점
- 카드 제거/카드 강화 같은 서브 화면
- 결과 메시지

## 화면 구성

현재 핵심 화면은 8개다.

1. 메인 메뉴
2. 맵
3. 전투
4. 카드 보상
5. 상점
6. 이벤트
7. 휴식
8. 런 결과

공통 UI 원칙:

- 상단에는 현재 목표와 다음 행동을 보여준다.
- 사용 가능한 핵심 버튼에는 `▶`와 강조 스타일을 준다.
- 불가능한 카드, 노드, 버튼은 어둡게 보여준다.
- 전투에서는 사용 가능한 카드, 공격 가능한 유닛, 공격 대상, 선택된 유닛을 서로 다르게 표시한다.
- 빌드 정보는 장식이 아니라 런 방향을 알려주는 핵심 정보로 취급한다.

## 현재 제외 범위

아래 기능은 현재 프로젝트 범위에서 제외한다.

- PvP
- 랭크전
- 시즌 운영
- 카드팩 수집 루프
- 온라인 저장 중심 플레이
- 카드/유물 해금 체계

## 남은 과제

- 완전한 트리형 분기 맵과 경로 선택 구조
- 카드/유물 효과의 밸런스 조정
- 저주/의식 스택의 장기 빌드 시너지 확장
- 사운드, 파티클, 전투 애니메이션 폴리싱
- 카드/유물/이벤트 설명의 용어 통일
- 자동 테스트에서 발생하는 Godot 종료 시 리소스 leak warning 정리
