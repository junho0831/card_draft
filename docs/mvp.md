# Card Draft MVP

## 목표

`Card Draft`의 현재 목표는 온라인 TCG 검증이 아니라, `빌드 완성형 싱글플레이 로그라이크 덱빌딩 게임`의 코어 루프를 Godot 안에서 완성하는 것이다.

핵심은 카드 수집보다 빌드 완성이다.

- 사망 빌드
- 드로우 빌드
- 버프 빌드
- 화염 피해 빌드
- 저체력 역전 빌드
- 소환 빌드

## 현재 범위

- 메인 메뉴에서 `새 런 시작`, `이어하기`, `메타 강화`, `카드 도감`
- 시작 종족 선택 없음
- Act 2개
  - 국경지대
  - 죽음의 성
- 노드 흐름
  - 일반전투
  - 일반전투
  - 이벤트
  - 일반전투
  - 상점
  - 엘리트
  - 휴식
  - 보스
- 전투 승리 후 카드 3장 제시, 1장 선택
- 런 전용 상점, 이벤트, 휴식
- 보스 승리 시 최대 체력 +5
- 영혼석과 메타 업그레이드
- 빌드 태그 UI와 활성 효과
- 초보자 유도 UI
  - 메인 메뉴, 맵, 전투, 보상에서 다음 행동을 직접 표시
  - 클릭 가능한 핵심 대상은 밝게, 현재 불가능한 대상은 어둡게 표시
- 런 저장/이어하기

## 콘텐츠

- 스타터 카드 4종
- 런 카드 풀 30장
- 유물 15개
- 이벤트 5개
- 일반 적 6종
- 엘리트 2종
- 보스 2종

## 전투 규칙

- 영웅 체력 0이 되면 패배
- 적 영웅 체력을 0으로 만들면 승리
- 저주/의식 같은 특수 스택은 카드 효과로만 남기고 승리 조건으로 사용하지 않는다.
- 마나는 1부터 시작, 턴마다 +1, 최대 10
- 필드는 플레이어/적 각각 5칸
- 유닛 전투는 동시 피해

## 빌드 태그

카드와 유물은 `build_tags` 배열로 빌드 방향을 가진다.

- `fire`: 화염 피해 +2
- `draw`: 플레이어 턴 시작 시 카드 1장 추가 드로우
- `death`: 아군 유닛 사망 시 적 영웅 피해 1
- `buff`: 아군 유닛 소환 시 체력 +1
- `low_hp`: 내 영웅 체력 50% 이하일 때 내 유닛 공격 피해 +1
- `summon`: 전투 시작 시 1/1 중립 토큰 1장 소환

활성 기준은 현재 빌드 점수 5 이상이다. 카드 보상 3장 중 1장은 현재 최고 빌드 태그와 맞는 카드에서 우선 제시한다.

## 시각적 유도 원칙

게임은 단순하게 읽혀야 한다. 플레이어가 규칙을 외우기 전에 화면만 보고 다음 행동을 알 수 있어야 한다.

- Visual Guidance
  - 모든 핵심 화면에는 `다음 행동` 배너를 둔다.
  - 맵에서는 현재 진입 가능한 노드만 밝게 보이고, 완료/잠김 노드는 어둡게 보인다.
  - 보상 화면에서는 선택해야 할 카드 3장이 중앙에 가장 크게 보여야 한다.
- Affordance
  - 누를 수 있는 핵심 버튼에는 `▶`를 붙인다.
  - 전투 중 사용 가능한 카드는 밝은 테두리와 `사용 ▶` 문구를 사용한다.
  - 공격 가능한 유닛은 `공격 가능`, 선택한 유닛은 `선택됨`, 공격 대상은 `공격 대상`으로 표시한다.
- Player Guidance
  - 전투 안내 문구는 현재 상태에 따라 바뀐다.
  - 예: 카드 사용 가능, 공격 대상 선택, 영웅 공격 가능, 적 턴 대기, 턴 종료 필요.
  - 안내는 긴 튜토리얼 문장보다 짧은 행동 문장으로 쓴다.
- Visual Hierarchy
  - 전투 목표는 항상 `적 영웅 처치`로 고정해서 가장 먼저 보이게 한다.
  - 적 영웅 HP는 일반 텍스트보다 크게 표시한다.
  - 보상 화면의 추천 카드는 금색 테두리와 `현재 빌드 추천` 문구를 사용한다.
  - 보조 버튼은 어둡게 두고, 다음 진행 버튼만 강하게 표시한다.

## 전투 UX 기준

전투 화면은 아래 순서로 시선을 유도한다.

1. 현재 목표: 적 영웅 체력 0
2. 다음 행동: 카드 사용, 유닛 선택, 공격 대상 선택, 턴 종료
3. 적 영웅 HP와 필드
4. 내 필드와 플레이어 상태
5. 현재 빌드 점수와 활성 빌드
6. 마나, 손패, 전투 로그

전투 중 빌드 점수는 장식 정보가 아니라 핵심 정보다. 플레이어가 `이번 런은 어떤 빌드인가`를 계속 확인할 수 있어야 하므로, 전투 화면에서도 숨기지 않는다.
