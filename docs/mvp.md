# Card Draft MVP

## 목표

`Card Draft`는 판타지마스터즈에서 영감을 받은 필드 전투 중심 온라인 TCG 프로토타입이다. MVP의 목적은 거래, 카드팩, 길드, 관전 같은 확장 기능을 넣기 전에 종족 시너지, 속성 정체성, 유닛 전투, 덱 빌딩이 실제로 재미있는지 검증하는 것이다.

## 핵심 규칙

- 승리 조건: 상대 영웅 체력 20을 0으로 만든다.
- 마나: 첫 턴 1마나로 시작하고, 매 턴 1씩 증가하며, 최대 10까지 오른다. 매 턴 시작 시 현재 최대 마나만큼 회복한다.
- 필드: 플레이어당 유닛 최대 5장.
- 전투: 유닛이 유닛을 공격하면 양쪽 유닛이 동시에 피해를 준다.
- 덱: 최종 상품 규칙 기준 30장, 동일 카드 최대 3장.
- MVP 카드 종류: 유닛, 주문, 장착.

## 종족 방향

- 인간: 안정적인 능력치와 아군 버프.
- 엘프: 저코스트 전개와 카드 드로우.
- 언데드: 사망 이득, 부활, 희생 효과.
- 중립: 모든 덱에서 사용할 수 있는 기본 유틸리티.

## 속성 방향

- 화염: 직접 피해.
- 물: 회복.
- 바람: 기동성 또는 추가 공격.
- 대지: 체력과 방어.
- 암흑: 희생과 자해.
- 빛: 버프와 지원.

## 현재 프로토타입 범위

- Godot 로컬 MVP 프로토타입.
- 메인 메뉴, 모드 선택, 덱 구성, 카드 보관함, 설정, 보상 화면.
- 일반전/랭크전 선택 후 AI 매칭 연출.
- `res://data/cards.json` 기반 카드 데이터와 로컬 보유 카드 데이터.
- 저장된 30장 덱으로 전투 시작.
- 손패, 마나, 필드, 유닛 공격, 영웅 공격, 주문, 장착, 전투 연출, 사망 처리, 기본 승패 처리.
- 골드, 카드 보상, 랭크 점수 로컬 진행.

## 시즌1 상품 범위

- 총 60장: 인간 15장, 엘프 15장, 언데드 15장, 중립 15장.
- 일반전 PvP.
- 브론즈, 실버, 골드, 플래티넘, 다이아 랭크전.
- 퀘스트 보상과 승리 보상을 통한 카드 획득.

## 시즌1 제외 범위

- 카드 강화.
- 카드 진화.
- 거래소 또는 경매장.
- 길드.
- 친구 시스템.
- 관전.
- 카드팩.

## 목표 기술 구조

- 클라이언트: Godot.
- 서버: Spring Boot.
- 데이터베이스: PostgreSQL.
- 캐시와 매칭/세션 상태: Redis.

## 구현 순서

1. 로컬 Godot 프로토타입에서 결정론적인 게임 규칙을 고정한다.
2. 카드 정의를 외부 JSON 리소스로 분리한다. 현재 샘플 카드셋은 완료된 상태다.
3. 30장 덱과 동일 카드 최대 3장 제한을 검증하는 덱 빌더를 추가한다. 현재 로컬 MVP에 구현된 상태다.
4. 로컬 메뉴, AI 매칭 연출, 보상, 랭크 점수를 연결한다. 현재 로컬 MVP에 구현된 상태다.
5. Spring Boot 게임 세션 API를 구현한다.
6. Redis 기반 매칭과 재접속 가능한 세션 상태를 추가한다.
7. PostgreSQL에 카드 인벤토리와 보상 데이터를 저장한다.
